## 一、执行摘要

SZZKLLM 经过 v1.0（基线）、v1.1（CoT 扩充）、v1.2（干净数据+CoT）三轮迭代，v1.2 在 200 题 benchmark 上达到 **61.0%**，但仍有 **72 道题持续错误**（含 58 道顽固错误所有版本全错）。

本报告对这 72 道错误进行系统性根因分析，分为三大根因类别、六条具体根因链，并给出修复优先级和预期收益。

### 核心结论

```
72 道打开错误中:
├── 52 道 (72%) 是 easy 基础送分题 → 最高性价比修复
├── 38 道 (53%) 根因是知识缺失 → 需补充针对性训练数据
├── 32 道 (44%) 根因是逻辑推理缺陷 → 需增强推理能力
└── 14 道 (19%) 伴随格式输出问题(pred=?) → CoT副作用
```

**最大发现**：模型在 52 道基础送分题上反复犯错，这些题的正确率应该趋近 100%，但实际上模型连绝对值、轴对称、纯净物等最基础概念都无法稳定掌握。

## 三、根因分类体系

### 3.1 一级根因：知识缺失

**38 题 (53%)**，核心子类：

```
concept_misunderstanding (22):  ████████████████████████
  → 模型对概念的理解有偏差，例如：
    - 将绝对值理解为有符号数 (#12)
    - 混淆轴对称图形与一般四边形 (#17)
    - 不理解纯净物与混合物的区别 (#136)
    - 混淆公民与人民的概念 (#163, #185)

fact_error (11):               ███████████
  → 模型记忆了错误的事实知识：
    - 圆周长公式 vs 面积公式混淆 (#26)
    - 课文朗读节奏划分错误 (#169, #198)

formula_misapplication (2):    ██

```

**根因链**：训练数据中缺乏足够的基础概念辨析题 → 模型从未见过这些概念的正反例对比 → 选择题中随机猜测或偏向频率高的选项。

### 3.2 一级根因：逻辑缺陷

**32 题 (44%)**，核心子类：

```
trap_fall (18):                ██████████████████████
  → 模型被干扰项误导，例如：
    - 历史题选项看似都合理，但忽略了限定条件 (#68, #74, #84, #89)
    - 政治题把"公民"和"人民"的干扰项混淆 (#163)

reasoning_chain_error (12):    ████████████
  → 推理过程存在断链：
    - 一次函数图像无法正确组合k<0和b>0的象限 (#27)
    - 幂运算中混淆乘法与指数规则 (#19)

condition_miss (2):            ██
```

**根因链**：CoT 训练让模型产生了"推理幻觉"——看似在推理，但推理过程本身包含错误 → 模型信心很高但选出错误答案。

### 3.3 一级根因：格式/输出问题

**2 题直接归类，但审核发现另外 14 题 `pred=?` 的本质也是格式问题**

```
cot_unparseable (至少14题实际):   ██████████████
  → 模型输出冗长CoT文本，答案提取器无法解析最终答案
  → 这14题被 DeepSeek 最初错标为 logic/knowledge
  → 审核纠正为 format/cot_unparseable

context_noise (1):              █
```

**根因链**：v1.2 保留了 CoT 训练 → 模型对选择题也生成【分析】+【答案】格式 → 部分输出中【答案】缺失或格式不规范 → 答案提取器 fail → pred=? → 被记作错误（但模型可能"心里"知道答案）。

## 六、v1.3 训练建议

### 6.1 数据策略

```
v1.3 训练集构成:
├── v1.2 清洗后基础数据      430 条 (保留)
├── 基础概念辨析增强          200 条 (新增，覆盖7学科)
│   └── 每道题: 原题 + 概念解释 + 正反例对比
├── 陷阱变体训练              120 条 (新增)
│   └── 原题 + 人为设置相似干扰项版本
├── 公式/事实记忆卡           100 条 (新增)
│   └── 简洁QA格式: "绝对值是什么？|-3|=3"
└── 优化后的CoT推理           50 条 (精选，非全量)
    └── 仅对需要推理的题目，不强制所有题都CoT
总计: ~900 条
```

### 6.2 训练策略调整

1. **选择性 CoT**：不要在选择题上强制 CoT。训练数据中混入"直接答"和"带推理"两种格式，让模型学会计情况选择。
2. **输出格式指令**：在 system prompt 中加入选择题专用指令："对于选择题，请先输出分析过程，最后以'【答案】X'结尾"。
3. **减少噪声**：替换 2 道已知噪声题 (#11, #134) 为干净版本。

### 6.3 答案提取器优化

当前 `extract_answer()` 对 CoT 输出的末尾答案提取不够强健。14 题 `pred=?` 中，部分可能是提取器问题而非模型真正错误。

**建议**：
- 增加 `【答案】X` 模式的权重（已有，但可能匹配顺序太靠后）
- 对末尾 50 字符做更激进的字母匹配
- 考虑用正则提取后，对比 benchmark 标准答案做二次校验
